SECTION 1 — PROJECT OVERVIEW
──────────────────────────────
This is a quantitative portfolio optimization engine for Vietnamese stocks
listed on HOSE/HNX. Given a user's target annual return r*, the system
computes the minimum-variance portfolio that hits that target — i.e., the
portfolio with the lowest possible risk for the investor's desired return.

The mathematical stack is built entirely in linear algebra:
  • EWMA covariance estimation  (exponentially-weighted moving average)
  • Eigendecomposition           Σ = QΛQᵀ
  • James-Stein μ shrinkage      reduces estimation error in expected returns
  • L2 / Tikhonov regularization Σ_reg = Σ + λI  (prevents extreme weights)
  • Quadratic Program (QP)       min wᵀΣ_reg w  s.t. μᵀw = r*, 1ᵀw = 1, 0≤wᵢ≤0.15

The optimizer is NOT a black-box ML model. Every number is traceable back to
a closed-form linear algebra formula — this is an explicit design choice.

Do NOT integrate any external "AI prediction" for expected returns. The
expected returns μ are computed solely from EWMA of historical log-returns.

SECTION 4 — API CONTRACT (what the backend exposes)
──────────────────────────────────────────────────────
The UI should call a thin wrapper around the optimizer. The interface is:

  INPUT  (from user):
    r_target  : float  — target annual return (e.g. 0.15 for 15%)

  OUTPUT (from optimizer — all derived on-the-fly from processed/ artifacts):
    weights   : dict[ticker → float]      portfolio weights (sum to 1)
    port_vol  : float                     annualized portfolio volatility
    port_ret  : float                     annualized expected return
    sharpe    : float                     (port_ret − RFR) / port_vol
    n_eff     : float                     effective number of stocks (1/HHI)
    zone      : str                       "reliable" | "caution"
    ret_gmv   : float                     min achievable return (lower bound)
    ret_max   : float                     max reliable-zone return (upper bound)
    frontier  : list[{ret, vol, sharpe}]  full efficient frontier points

  VALIDATION:
    The UI must enforce:    ret_gmv ≤ r_target ≤ ret_max_caution
    Display warning if:     r_target > ret_reliable_max  (caution zone)
    Hard reject if:         r_target < ret_gmv  OR  r_target > ret_max_caution

  The optimizer enforces per-stock cap (≤15%) and long-only (wᵢ ≥ 0)
  automatically — no need to re-validate these in the UI.


SECTION 5 — RELIABLE ZONE vs CAUTION ZONE
───────────────────────────────────────────
This is the single most important concept for UI communication:

  N_eff = 1 / Σwᵢ²   (effective number of positions, Herfindahl-Hirschman)

  • N_eff ≥ 3  →  RELIABLE ZONE   (optimizer is well-conditioned)
  • N_eff < 3  →  CAUTION ZONE    (portfolio is concentrated; math is correct
                                   but input μ uncertainty dominates)

The math (QP, eigendecomposition) is ALWAYS correct. The caution zone flag
means the estimated expected returns (μ) are unreliable for high targets,
NOT that the portfolio calculation itself has a bug.

  ret_reliable_max  = highest r_target where N_eff ≥ 3
  ret_caution_max   = maximum r_target the optimizer can reach (hard limit)

Suggested UI language:
  Reliable zone:  "Portfolio is well-diversified. Proceed with confidence."
  Caution zone:   "High return target leads to concentrated positions.
                   Historical returns may not reflect future performance."

SECTION 7 — KNOWN LIMITATIONS (communicate to end user)
─────────────────────────────────────────────────────────
1. BACKWARD-LOOKING μ: Expected returns are purely based on EWMA of past
   log-returns. They do not incorporate analyst forecasts, macro signals,
   or forward-looking data. Past performance ≠ future results.

2. ARITHMETIC ANNUALIZATION: Daily μ_annual = daily_mean × 252.
   This is a linear approximation. For log returns over long horizons,
   geometric compounding (exp(252·μ_daily) − 1) is more precise but was
   intentionally kept simple to match the linear algebra scope.

3. UNIVERSE IS FIXED: Only these 39 tickers. No dynamic inclusion/exclusion.

4. COVARIANCE USES 2018–2026 DATA: EWMA half-life = 252 days means data
   from 2018 has < 0.01% weight. The "ghost regime" of old market structure
   is effectively eliminated. But the 2020 COVID crash is still in the window.

5. TRANSACTION COSTS NOT MODELLED: The optimizer ignores bid-ask spreads,
   brokerage fees (typ. 0.15–0.25% on Vietnamese exchanges), or turnover.
   For a practical tool, add a turnover constraint or cost term.

6. THIS IS NOT FINANCIAL ADVICE. Present prominent disclaimer in the UI.

  With old formula: alpha ≈ 0.99 (almost no shrinkage — JS was a no-op).
  With correct formula: alpha → 0 (full shrinkage, potential degeneracy).
